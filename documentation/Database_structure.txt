Database Structure Documentation
Overview
This database powers an e-commerce application with products, categories, and their relationships. The schema supports hierarchical categories, product variations, and flexible filtering capabilities.

Core Tables
1. Products Table (public.products)
Purpose: Stores product information including pricing, inventory, and media.

sql
CREATE TABLE public.products (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
  name TEXT NOT NULL,
  price INTEGER NOT NULL,           -- Actual selling price
  discount_price INTEGER NULL,                    -- Discount price for holidays etc
  description TEXT NULL,
  stock_quantity INTEGER NULL,           -- Total inventory across all variants
  gender_target TEXT NULL,               -- 'men', 'women', 'unisex', 'kids'
  image_urls TEXT[] NOT NULL,           -- Array of image URLs
  video_urls TEXT[] NULL,               -- Array of video URLs
  is_featured BOOLEAN NULL DEFAULT false,
  is_active BOOLEAN NULL DEFAULT true,  -- Soft delete flag
  rating INTEGER NULL,                  -- Average rating (0-5)
  
  -- Filtering attributes (array-based for simplicity)
  sizes TEXT[] NULL,                     -- Available sizes: ['S', 'M', 'L']
  colors TEXT[] NULL,                    -- Available colors: ['red', 'blue']
  brand TEXT NULL,                       -- Brand name
  
  CONSTRAINT products_pkey PRIMARY KEY (id)
) TABLESPACE pg_default;
Key Fields:

discount_price vs price: Discount vs actual selling price

image_urls/video_urls: Arrays for multiple media items

sizes/colors: Arrays to support multiple options

is_active: Soft deletion support

2. Categories Table (public.categories)
Purpose: Hierarchical product categorization system.

sql
CREATE TABLE public.categories (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
  name TEXT NOT NULL,                    -- Display name
  slug TEXT NOT NULL,                    -- URL-friendly identifier
  description TEXT NULL,
  parent_id BIGINT NULL,                 -- For hierarchical structure
  image_url TEXT NULL,                   -- Category banner/image
  display_order INTEGER NULL DEFAULT 0,  -- Sorting within parent
  is_active BOOLEAN NULL DEFAULT true,   -- Active status
  
  -- SEO fields
  meta_title TEXT NULL,
  meta_description TEXT NULL,
  
  CONSTRAINT categories_pkey PRIMARY KEY (id),
  CONSTRAINT categories_slug_key UNIQUE (slug),
  CONSTRAINT categories_parent_id_fkey FOREIGN KEY (parent_id) 
    REFERENCES categories (id) ON DELETE SET NULL
) TABLESPACE pg_default;
Hierarchy Example:

text
Electronics (id: 1, parent_id: NULL)
├── Mobile Phones (id: 2, parent_id: 1)
└── Laptops (id: 3, parent_id: 1)
    └── Gaming Laptops (id: 4, parent_id: 3)
Indexes:

idx_categories_slug: Fast slug-based lookups

idx_categories_parent_id: Efficient hierarchy queries

idx_categories_display_order: Sorting optimization

3. Junction Table (public.product_categories)
Purpose: Many-to-many relationship between products and categories.

sql
CREATE TABLE public.product_categories (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL,
  product_id BIGINT NOT NULL,
  category_id BIGINT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
  is_primary BOOLEAN NULL DEFAULT false,  -- Marks main category
  
  CONSTRAINT product_categories_pkey PRIMARY KEY (id),
  CONSTRAINT product_categories_product_id_fkey 
    FOREIGN KEY (product_id) REFERENCES products (id) ON DELETE CASCADE,
  CONSTRAINT product_categories_category_id_fkey 
    FOREIGN KEY (category_id) REFERENCES categories (id) ON DELETE CASCADE,
  CONSTRAINT product_categories_unique 
    UNIQUE (product_id, category_id)
) TABLESPACE pg_default;

-- Performance indexes
CREATE INDEX idx_product_categories_product_id 
  ON public.product_categories USING BTREE (product_id);
  
CREATE INDEX idx_product_categories_category_id 
  ON public.product_categories USING BTREE (category_id);
Relationships:

One product → multiple categories

One category → multiple products

ON DELETE CASCADE: Auto-cleanup when products/categories are deleted
