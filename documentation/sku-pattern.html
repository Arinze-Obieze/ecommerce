SKU Pattern

Yes.

The SKU pattern is:

ZVA-<STORE4>-<PROD4>-<SEQ4>

How Each Part Works

  • ZVA: fixed marketplace prefix
  • <STORE4>: first 4 uppercase alphanumeric characters from the store slug; if that is missing, it falls back to the store name
  • <PROD4>: first 4 uppercase alphanumeric characters from the product slug; if that is missing, it falls back to the product name
  • <SEQ4>: a 4-digit sequence number padded with zeros, incremented per matching store/product prefix

Examples

Store slug zara, product slug shirt-men -> ZVA-ZARA-SHIR-0001
Store slug nike-outlet, product slug air-max-90 -> ZVA-NIKE-AIRM-0001
Next product with the same store/product prefix becomes ...-0002

Implementation Details

  • Non-alphanumeric characters are stripped before building the codes.
  • If a store or product code is shorter than 4 characters, it is padded with X.
  • The sequence is determined by checking existing SKUs with the same ZVA-STORE4-PROD4- prefix and adding 1.

So yes, it follows both store and product.
